Insufficient Session Expiration

This rule logs audit points when finding not-expiring HttpSession objects. This can make some attacks more successful. If an attacker can intercept a session id, it will be reusable at any time. Expiring the session at some point, reduces the window of session-related attacks.

Example: 

[...]
protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
    HttpSession s = req.getSession(true);
    s.setMaxInactiveInterval(-1); // Indicates the session should never timeout.
}
[...]
