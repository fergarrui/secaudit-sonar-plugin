XSS finder

This rule logs audit points when detecting a potential XSS. Data written to the response output must be properly validated if it comes from an untrusted source.

Logs issues when writing to a PrintWriter or OutputStream coming from javax.servlet.ServletResponse#getWriter() or javax.servlet.ServletResponse#getOutputStream():

  • java.io.PrintWriter#print()
  • java.io.PrintWriter#println()
  • java.io.PrintWriter#write()
  • java.io.OutputStream#write()

    • Example: 

    class A extends HttpServlet {
  @Override
  void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
    resp.getWriter().print(req.getParameter("httpParameter")); // XSS found
  }
}

    See

    • OWASP XSS Prevention.
    • WASC Cross Site Scripting.
    • OWASP Top 10 XSS.
    • CWE-79 - Improper Neutralization of Input During Web Page Generation ('Cross-site Scripting').
