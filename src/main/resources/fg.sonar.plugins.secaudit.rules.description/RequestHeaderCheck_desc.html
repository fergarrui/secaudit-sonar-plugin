Request Header Check

This rule logs audit points when retrieving header values (Referer, cookies, user-agent, etc). Treat any decision taken from this value like a malicious input.

Example: 

    [...]
    public void doGet(HttpServletRequest request, HttpServletResponse response) {
        String header = request.getHeader("AnyHeader"); // Potentially dangerous
        Cookie[] cookies = request.getCookies(); // Potentially dangerous
    }
    [...]

See

  • Http header fields.
  • CWE-807 Reliance on Untrusted Inputs in a Security Decision
  • CWE-93 Improper Neutralization of CRLF Sequences ('CRLF Injection')
  • CWE-113 Improper Neutralization of CRLF Sequences in HTTP Headers ('HTTP Response Splitting')
  • CWE-784 Reliance on Cookies without Validation and Integrity Checking in a Security Decision
  • CWE-293 Using Referer Field for Authentication
